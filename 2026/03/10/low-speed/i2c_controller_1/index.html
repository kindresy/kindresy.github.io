寄存器详解

IC_CON - 控制寄存器

IC_CON是DW_apb_i2c的核心控制寄存器,用于配置控制器的基本工作模式:

关键位域

  • MASTER_MODE (bit 0): 主机模式使能

    • 1 = 主机模式
    • 0 = 从机模式

  • SPEED (bits 2:1): 速度模式选择

    • 00 = 标准速度(100 kb/s)
    • 01 = 快速速度(400 kb/s)
    • 10 = 高速模式(3.4 Mb/s)

  • IC_10BITADDR_MASTER (bit 12): 主机地址模式

    • 1 = 10-bit地址模式
    • 0 = 7-bit地址模式

  • IC_10BITADDR_SLAVE (bit 3): 从机地址模式

    • 1 = 10-bit地址模式
    • 0 = 7-bit地址模式

  • IC_SLAVE_DISABLE (bit 6): 从机模式禁用

    • 1 = 禁用从机模式
    • 0 = 使能从机模式

动态地址更新

I2C_DYNAMIC_TAR_UPDATE参数决定了地址更新时机:

I2C_DYNAMIC_TAR_UPDATE = 0 (静态模式)

  • 切换目标地址时需要:禁用控制器→重配置IC_TAR→重新启用
  • 流程复杂,但配置稳定

I2C_DYNAMIC_TAR_UPDATE = 1 (动态模式)

  • 可在运行中直接修改IC_TAR,无需禁用控制器
  • IC_10BITADDR_MASTER位可动态调整
  • 适合需要频繁切换从机地址的场景

IC_TAR - 目标地址寄存器

用于指定主机模式下要访问的从机地址:

  • IC_TAR (bits 9:0): 目标从机地址
  • GC_OR_START (bit 10): 通用呼叫或START BYTE命令

IC_DATA_CMD - 数据命令寄存器

这是最重要的数据传输寄存器,所有读写操作都通过它完成:

寄存器结构

根据IC_EMPTYFIFO_HOLD_MASTER_EN配置,寄存器结构有所不同:

IC_EMPTYFIFO_HOLD_MASTER_EN = 0

  • CMD (bit 8): 传输类型
    • 1 = 读操作
    • 0 = 写操作
  • DATA (bits 7:0): 数据区

IC_EMPTYFIFO_HOLD_MASTER_EN = 1

  • RESTART (bit 10): 生成RESTART信号
  • STOP (bit 9): 生成STOP信号
  • CMD (bit 8): 传输类型
  • DATA (bits 7:0): 数据区

IC_ENABLE - 使能寄存器

  • IC_ENABLE (bit 0): 控制器使能

    • 1 = 使能控制器
    • 0 = 禁用控制器

  • ABORT (bit 1): 强制中止传输

    • 仅主机模式有效
    • 触发后发送STOP条件,清空Tx FIFO

IC_ENABLE_STATUS - 使能状态寄存器 (v2.03a+)

  • IC_EN (bit 0): 控制器实际使能状态
    • 用于判断硬件是否已完成关闭

时钟配置寄存器组

IC_FS_SCL_LCNT / IC_FS_SCL_HCNT

配置Fast Mode下的SCL低电平和高电平周期:

  • 最小值约束:
    • LCNT > SPKLEN + 7 (IC_CLK_FREQ_OPTIMIZATION = 0)
    • LCNT ≥ 6 (IC_CLK_FREQ_OPTIMIZATION = 1)
    • HCNT > SPKLEN + 5 (IC_CLK_FREQ_OPTIMIZATION = 0)
    • HCNT ≥ 1 (IC_CLK_FREQ_OPTIMIZATION = 1)

IC_FS_SPKLEN - 毛刺抑制寄存器

配置Fast Mode下的最大毛刺抑制时间:

  • 最小值为1
  • 计算公式: SPKLEN = 毛刺时间 / ic_clk周期 - 1

IC_SDA_HOLD - SDA保持时间寄存器

控制SDA数据保持时间(tHD;DAT):

  • IC_SDA_RX_HOLD (bits 15:0): 接收模式SDA保持时间
  • IC_SDA_TX_HOLD (bits 31:16): 发送模式SDA保持时间
  • 单位: 1个ic_clk周期
  • 必须在控制器禁用时写入

DMA配置寄存器

IC_DMA_CR - DMA控制寄存器

  • TDMAE (bit 1): 发送DMA使能
  • RDMAE (bit 0): 接收DMA使能

IC_DMA_TDLR - 发送DMA水位寄存器

设置TX FIFO触发DMA请求的水位阈值:

  • 数值越大,触发越早,FIFO溢出风险越低
  • 数值越小,触发越晚,总线效率越高但下溢风险增加

IC_DMA_RDLR - 接收DMA水位寄存器

设置RX FIFO触发DMA请求的水位阈值:

  • 最优配置: SRC_MSIZE = RDLR + 1

Programming Flow (编程流程)

主机模式初始化流程

静态模式初始化 (I2C_DYNAMIC_TAR_UPDATE = 0)

  1. 禁用控制器: 向IC_ENABLE[0]写入0
  2. 配置控制寄存器:
    • 写入IC_CON,设置速度模式、地址模式
    • 确保IC_SLAVE_DISABLE = 1, MASTER_MODE = 1
  3. 设置目标地址: 写入IC_TAR,指定从机地址
  4. 高速模式配置 (可选): 写入IC_HS_MADDR
  5. 启用控制器: 向IC_ENABLE[0]写入1
  6. 发起传输: 向IC_DATA_CMD写入数据/命令

切换从机地址时需重复步骤1-6

动态模式初始化 (I2C_DYNAMIC_TAR_UPDATE = 1)

  1. 禁用控制器 (仅初始化时): 向IC_ENABLE[0]写入0
  2. 配置控制寄存器:
    • 写入IC_CON,设置速度模式
    • 主机地址模式由IC_10BITADDR_MASTER动态控制
  3. 设置目标地址: 写入IC_TAR
  4. 高速模式配置 (可选): 写入IC_HS_MADDR
  5. 启用控制器: 向IC_ENABLE[0]写入1
  6. 发起传输: 向IC_DATA_CMD写入数据/命令
  7. 切换地址 (运行中): 直接写入新的IC_TAR,无需禁用

数据传输流程

Master Transmitter (主机发送器)

IC_EMPTYFIFO_HOLD_MASTER_EN = 0

  1. 向Tx FIFO写入数据(CMD=0)
  2. 控制器自动产生START条件
  3. 发送地址+W+数据字节
  4. Tx FIFO为空时自动生成STOP条件

IC_EMPTYFIFO_HOLD_MASTER_EN = 1

  1. 向Tx FIFO写入数据(CMD=0)
  2. 控制器产生START,开始传输
  3. 最后一个字节需设置STOP位=1
  4. 否则控制器会拉低SCL,等待新数据

Master Receiver (主机接收器)

IC_EMPTYFIFO_HOLD_MASTER_EN = 0

  1. 向Tx FIFO写入读命令(CMD=1)
  2. 控制器产生START,发送地址+R
  3. 依次读取数据字节,最后发送NACK
  4. Tx FIFO为空时生成STOP条件

IC_EMPTYFIFO_HOLD_MASTER_EN = 1

  1. 向Tx FIFO写入读命令(CMD=1)
  2. 控制器发送地址+R,开始读取
  3. 最后一个命令需设置STOP位=1
  4. 否则控制器会拉低SCL保持总线

RESTART条件生成

当IC_DATA_CMD[10] = 1且IC_RESTART_EN = 1时:

发送器:

  • 当前字节传输完成后生成RESTART
  • 立即开始新传输(地址+W+数据)
  • FIFO为空时先拉低SCL,等新数据

接收器:

  • 必须先发送NACK
  • 然后生成RESTART,开始新读传输

控制器禁用流程

标准禁用流程

  1. 定义轮询间隔: t_i2c_poll = 最高I²C速率信号周期 × 10
  2. 定义超时参数: MAX_T_POLL_COUNT
  3. 阻塞新事务: 阻止软件发起新的I²C主机事务
  4. 初始化计数器: POLL_COUNT = 0
  5. 发起禁用: 向IC_ENABLE[0]写入0
  6. 轮询状态:
    • 读取IC_ENABLE_STATUS[0]
    • POLL_COUNT++
    • 若POLL_COUNT >= MAX_T_POLL_COUNT,返回错误
  7. 等待完成:
    • 若IC_EN = 1,休眠t_i2c_poll,回到步骤6
    • 若IC_EN = 0,返回成功

强制中止流程 (ABORT)

用于IC_EMPTYFIFO_HOLD_MASTER_EN = 1且需立即禁用的情况:

  1. 停止填充Tx FIFO
  2. 禁用DMA (可选): TDMAE置0
  3. 发起ABORT: 向IC_ENABLE[1]写入1
  4. 等待中断: 等待M_TX_ABRT中断
  5. 检查中止源: 读取IC_TX_ABRT_SOURCE,确认为ABRT_USER_ABRT

时钟配置流程

Fast Mode配置步骤

  1. 配置速率模式寄存器
  2. 确保ic_clk ≥ 32 MHz
  3. 配置IC_CON寄存器: 切换到Fast Mode或Fast Mode Plus
  4. 配置计数寄存器:
    • IC_FS_SCL_LCNT (低电平周期)
    • IC_FS_SCL_HCNT (高电平周期)
  5. 编程IC_FS_SPKLEN: 配置毛刺抑制时间
  6. 编程IC_SDA_SETUP: 满足最小数据建立时间

DMA配置流程

发送DMA配置

1. 配置DMA控制器:

  • BLOCK_TS: 总数据块大小
  • DEST_MSIZE: 单次突发长度

2. 配置I²C DMA水位:

  • IC_DMA_TDLR: 发送水位阈值
  • 最优配置: DEST_MSIZE = FIFO_DEPTH - IC_DMA_TDLR

3. 使能发送DMA:

  • IC_DMA_CR.TDMAE = 1

接收DMA配置

1. 配置DMA控制器:

  • BLOCK_TS: 总数据块大小
  • SRC_MSIZE: 单次突发长度

2. 配置I²C DMA水位:

  • IC_DMA_RDLR: 接收水位阈值
  • 最优配置: SRC_MSIZE = IC_DMA_RDLR + 1

3. 使能接收DMA:

  • IC_DMA_CR.RDMAE = 1

DMA水位选择策略

高水位配置 (TDLR大):

  • 优点: FIFO下溢风险低,传输可靠
  • 缺点: 突发次数多,总线利用率低

低水位配置 (TDLR小):

  • 优点: 突发次数少,总线效率高
  • 缺点: FIFO下溢风险高

Bus Clear流程

SDA线被拉低恢复

  1. 主机最多发送9个SCL时钟脉冲
  2. 同时尝试驱动SDA为高电平
  3. 若SDA恢复为高,发送STOP条件
  4. 若第9个时钟后SDA仍为低,需硬件复位

SCL线被拉低

  • 唯一解决方案: 硬件复位整个I²C总线
  • SCL被拉低后无法产生时钟,协议无法继续

Ultra-Fast Mode配置

特点:

  • 单向传输: 仅支持主机写操作
  • 速率: DC ~ 5 MHz
  • 第9个时钟由主机强制驱动为高(NACK)
  • 使用推挽驱动器,非开漏

配置步骤:

  1. IC_ULTRA_FAST_MODE = 1
  2. 配置IC_UFM_SPKLEN
  3. 禁用读操作(CMD位固定为0)
